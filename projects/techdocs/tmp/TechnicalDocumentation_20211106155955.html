the LN command

The ln command is used to create links between files. See the examples of its usage below:

  • ln -s

    Creates a symbolic(soft) link(pointer) to a file. If the file is deleted the link stops working.

    
┌──(kali㉿aBANANA)-[~]
└─$ ln -s Documents/myfile.txt .

┌──(kali㉿aBANANA)-[~]
└─$ ls -l myfile.txt
lrwxrwxrwx 1 kali kali 20 Nov 6 06:51 myfile.txt -> Documents/myfile.txt

┌──(kali㉿aBANANA)-[~]
└─$ cat myfile.txt
aBANANA

┌──(kali㉿aBANANA)-[~]
└─$ rm Documents/myfile.txt

┌──(kali㉿aBANANA)-[~]
└─$ cat myfile.txt
cat: myfile.txt: No such file or directory

┌──(kali㉿aBANANA)-[~]
└─$

  • ln -P file1.txt filetwo.txt

    Make a hard link called 'filetwo.txt' to the underlying INODE entry for the file 'file1.txt'. If 'file1.txt' is deleted, 'file2.txt' will still contiue working.

    
┌──(kali㉿aBANANA)-[/]
└─$ touch tmp/myfile.txt

┌──(kali㉿aBANANA)-[/]
└─$ echo 'aBANANA' > tmp/myfile.txt

┌──(kali㉿aBANANA)-[/]
└─$ sudo ln -P tmp/myfile.txt . 
[sudo] password for kali:

┌──(kali㉿aBANANA)-[/]
└─$ ls -l myfile.txt
-rw-r--r-- 2 kali kali 8 Nov 6 07:05 myfile.txt

┌──(kali㉿aBANANA)-[/]
└─$ cat myfile.txt
aBANANA

┌──(kali㉿aBANANA)-[/]
└─$ rm tmp/myfile.txt

┌──(kali㉿aBANANA)-[/]
└─$ cat myfile.txt
aBANANA

┌──(kali㉿aBANANA)-[/]
└─$

the MV command

The mv command is used for the purposes of moving files between locations, as well as in the place of the 'Rename' command used in Windows for the renaming of files. See examples of its usage below:

  • mv locationone/file.txt locationtwo/file.txt

    Move file.txt from Source to Destination directory.

    ┌──(kali㉿aBANANA)-[/]
└─$ touch tmp/myfile.txt

┌──(kali㉿aBANANA)-[/]
└─$ ls
bin dev home initrd.img.old lib32 libx32 media opt root sbin sys usr vmlinuz
boot etc initrd.img lib lib64 lost+found mnt proc run srv tmp var vmlinuz.old

┌──(kali㉿aBANANA)-[/]
└─$ sudo mv tmp/myfile.txt /

┌──(kali㉿aBANANA)-[/]
└─$ ls
bin etc initrd.img.old lib64 media opt run sys var
boot home lib libx32 mnt proc sbin tmp vmlinuz
dev initrd.img lib32 lost+found myfile.txt root srv usr vmlinuz.old

  • mv firstfilename.txt secondfilename.txt

    Rename the SOURCE filename to DESTINATION filename.

    ┌──(kali㉿aBANANA)-[/]
└─$ ls
bin etc initrd.img.old lib64 media opt run sys var
boot home lib libx32 mnt proc sbin tmp vmlinuz
dev initrd.img lib32 lost+found myfile.txt root srv usr vmlinuz.old

┌──(kali㉿aBANANA)-[/]
└─$ sudo mv myfile.txt myotherfile.txt

┌──(kali㉿aBANANA)-[/]
└─$ ls
bin etc initrd.img.old lib64 media opt run sys var
boot home lib libx32 mnt proc sbin tmp vmlinuz
dev initrd.img lib32 lost+found myotherfile.txt root srv usr vmlinuz.old

┌──(kali㉿aBANANA)-[/]
└─$

the MKDIR command

The mkdir command is used to create directories in the linux file system, if they do not already exist. See examples of its usage below:

  • mkdir /tmp/dirone

    Create a single directory called 'dirone' in the tmp/ directory.

    
┌──(kali㉿aBANANA)-[/]
└─$ mkdir /tmp/dirone

  • mkdir /tmp/dirtwo /tmp/dirthree /tmp/dirfour

    More than one directory is created under the tmp/ directory with a single command.

    
┌──(kali㉿aBANANA)-[/]
└─$ mkdir /tmp/dirtwo /tmp/dirthree /tmp/dirfour

  • mkdir -p /tmp/dirfive/dirsix/dirseven

    Adding the -p option to the mkdir commands allow for the creating directories, in directories, in directories, etc.

    
┌──(kali㉿aBANANA)-[/]
└─$ mkdir -p /tmp/dirfive/dirsix/dirseven

    List of the created directories in the tmp/ directory:

    ┌──(kali㉿aBANANA)-[/]
└─$ ls -R /tmp/dir*
/tmp/dirfive:
dirsix

/tmp/dirfive/dirsix:
dirseven

/tmp/dirfive/dirsix/dirseven:

/tmp/dirfour:

/tmp/dirone:

/tmp/dirthree:

/tmp/dirtwo:

the RM command

The rm command is used to remove files from the system. It does not, unless specified, remove directories. See examples of its usage below:

  • rm /tmp/myfile.txt

    The single file, myfile.txt, is deleted from the tmp/ directory.

    
┌──(kali㉿aBANANA)-[/]
└─$ ls /tmp/myfile.txt
/tmp/myfile.txt

┌──(kali㉿aBANANA)-[/]
└─$ rm /tmp/myfile.txt

┌──(kali㉿aBANANA)-[/]
└─$ ls /tmp/myfile.txt
ls: cannot access '/tmp/myfile.txt': No such file or directory

  • rm /tmp/myfileone.txt /tmp/myfiletwo.txt /tmp/myfilethree.txt

    With a single command multiple files, myfileone.txt/myfiletwo.txt/myfilethree.txt, can be deleted.

    
┌──(kali㉿aBANANA)-[/]
└─$ touch tmp/myfile{one,two,three}.txt

┌──(kali㉿aBANANA)-[/]
└─$ ls /tmp/myfile*
/tmp/myfileone.txt /tmp/myfilethree.txt /tmp/myfiletwo.txt

┌──(kali㉿aBANANA)-[/]
└─$ rm /tmp/myfileone.txt /tmp/myfiletwo.txt /tmp/myfilethree.txt

┌──(kali㉿aBANANA)-[/]
└─$ ls /tmp/myfile*
ls: cannot access '/tmp/myfile*': No such file or directory

  • rm -rf /tmp/dirone/

    Normally the rm command is unable to delete directories, but with the -rf option added it deletes the directory, with all subdirectories and files therein.

    
┌──(kali㉿aBANANA)-[/]
└─$ rm /tmp/dirone/
rm: cannot remove '/tmp/dirone/': Is a directory

┌──(kali㉿aBANANA)-[/]
└─$ rm -rf /tmp/dirone/

┌──(kali㉿aBANANA)-[/]
└─$ ls /tmp/dirone/
ls: cannot access '/tmp/dirone/': No such file or directory

the RMDIR command

The rmdir command is used to delete directories if the are empty. See exmaples of its usage below:

  • rmdir /tmp/dir*

    Remove all directories starting with the letters 'dir' in the tmp/ directory. This does not work with sub directories, so only the directories starting with 'dir' in the name, that does not have subdirectories, are successfully deleted.

    
┌──(kali㉿aBANANA)-[/]
└─$ ls /tmp/dir*
/tmp/dirfive:
dirsix

/tmp/dirfour:

/tmp/dirthree:

/tmp/dirtwo:

┌──(kali㉿aBANANA)-[/]
└─$ rmdir /tmp/dir*
rmdir: failed to remove '/tmp/dirfive': Directory not empty

  • rmdir -p /tmp/dirfive/dirsix/dirseven/

    By adding the -p option to the rmdir command a full directory patth can be deleted. Note that this cannot be used to delete all subdirectories of a directory.

    
┌──(kali㉿aBANANA)-[/]
└─$ ls -R /tmp/dirfive
/tmp/dirfive:
dirsix

/tmp/dirfive/dirsix:
dirseven

/tmp/dirfive/dirsix/dirseven:

┌──(kali㉿aBANANA)-[/]
└─$ rmdir /tmp/dirfive/dirsix/dirseven
